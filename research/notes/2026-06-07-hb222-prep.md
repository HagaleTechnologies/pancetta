## Implementation surface

1. **New function** `refine_candidate_against_known_tones` (~80-150 LOC):
   - 3×4 = 12-position search grid
   - At each position, evaluate `known_strength_how_7` score
   - Return best-scoring position as refined CostasCandidate

2. **New score function** `known_strength_how_7` (~50 LOC):
   - For each symbol s_i (i ∈ 0..78):
     - Look up the known tone at position i
     - Read complex spectrogram value at (refined_hz, t0 + i*sym, known_tone)
     - Compute magnitude difference vs previous symbol's value at known_tone
     - Accumulate (sign-flipped sum = "reward smooth transitions")
   - Return negative sum (higher = better coherence)

3. **SearchGrid struct** (~20 LOC).
