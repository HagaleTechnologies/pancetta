## Ranked candidates

### 1. hb-261 — Costas-pilot per-symbol amplitude normalisation + adaptive fading blend (priority 0.50)

**Technique**: Use FT8's three known Costas sync sequences (at symbol indices
≈0, ≈36, ≈72) as pilot symbols to estimate the time-varying complex channel
gain H(t). For each decoded candidate: (a) extract the complex DFT bin
amplitude at each Costas symbol position — these amplitudes are ALREADY
computed in pancetta's sync search and can be re-read; (b) interpolate
linearly across all 79 symbol positions; (c) compute per-symbol SNR estimate
from the interpolated amplitude; (d) detect fading when the amplitude range
across the Costas triplet exceeds ~3 dB; (e) scale each data-symbol LLR by
the corresponding estimated amplitude (MMSE-style division), which corrects
for selective fading induced amplitude modulation; (f) blend the
amplitude-normalised LLRs with the original max-log LLRs proportionally to
the fading depth — so static channels receive no modification.

**Source**: FT2 DECODIUM Raptor Engine (ft2.it, released 2026-05-19).
The relevant source modules are `ft2_channel_est.f90` and
`get_ft2_bitmetrics.f90`, described in the DECODIUM reference manual.
The underlying technique is standard OFDM/FSK pilot-aided channel estimation
(Cavers 1991; Van de Beek 1995; Dong & Tong 2002 for interpolated CE).
The Raptor implementation adapts these to FT8's sparse Costas pilot grid.

**Measured evidence**: DECODIUM documentation reports +0.5 to +1.5 dB on HF
selective fading (multipath-spread) channels. Zero-dB cost on AWGN and
static (non-fading) channels, confirmed by the blend gate: if fading depth
< 3 dB the modified LLRs are not used.

**Applicability to pancetta**:
- Attach point: `pancetta-ft8/src/decode.rs`, after the existing LLR
  computation (post-hb-038 whitening) and before the LDPC solver.
- Data already available: the complex Costas-band amplitudes are computed
  during the 2-D sync sweep and stored on each `SyncCandidate`. No FFT
  re-run needed — just extract the three Costas-band amplitudes and
  interpolate.
- Scope: ~120 lines in `decode.rs` + a new `channel_estimation.rs` helper.
  Stateless per-candidate, no inter-slot state, zero risk to existing paths.
- The fading detection gate means this is safe to enable by default: on a
  static AWGN corpus (raw_530_full) it is inert; it only fires when the
  Costas triplet shows a real fading signature.
- Synergy: applies before LDPC, so it feeds into both the first-pass and any
  OSD/BICM-ID passes (hb-252) that follow — the gain compounds if fading
  is the dominant failure mode.

**License**: Algorithm described in DECODIUM public release notes and standard
DSP textbooks. No GPL source consulted; no clean-room firewall needed.

### 2. hb-262 — Subcode Ensemble Decoding (SCED / aSCED) for short LDPC (priority 0.35)

**Technique**: Run multiple BP decoders in parallel, each on a *subcode*
(or *affine subcode*) derived from the original LDPC code by discarding
rows of the parity-check matrix. Each subcode has a smaller minimum distance
and relaxed graph, so BP converges on codewords that defeat the full
parity-check matrix. A majority-vote or ML-selection criterion chooses among
the ensemble outputs. Crucially, SCED does NOT require the automorphism group
of the code (which is NP-hard to compute for general LDPC codes), removing
the main practical barrier of the earlier Automorphism Ensemble Decoding
(AED) approach.

**Sources**:
- arXiv:2501.11993, "Subcode Ensemble Decoding of LDPC Codes" (Jan 2025)
- arXiv:2604.06889, "Affine Subcode Ensemble Decoding" (Apr 2026) — extends
  SCED to affine subcodes (cosets), gaining 0.2–0.3 dB over plain SCED

**Measured evidence**:
- SCED (arXiv:2501.11993): beats standalone BP and AED on CCSDS AR4JA
  LDPC(128,64) at BER ≤ 1e-4; approaches ML within 0.2 dB at 16–32 paths.
- aSCED (arXiv:2604.06889): further +0.2–0.3 dB over SCED; near-ML with 64
  parallel paths on BCH, CCSDS, 802.11n, and 5G NR LDPC codes.
- FT8 LDPC(174,91) is structurally similar to the CCSDS codes tested.

**Applicability to pancetta**:
- Attach point: `pancetta-ft8/src/ldpc.rs`. Replace or supplement the
  current single-path BP with an N-path SCED ensemble. The existing
  `LdpcConfig::max_iterations` budget is per-path; total iterations =
  N × per-path, so the tradeoff is controllable.
- FT8's automorphism group is NOT publicly known (unlike 5G/802.11n whose
  groups are standardised). SCED's avoidance of automorphism computation
  makes it the practical path for FT8.
- Synergy: relates to hb-257 (ensemble decoding); SCED is a strict
  improvement over the "parallel constituent decoders + ML selection"
  approach in hb-257 because the subcode construction is principled rather
  than ad-hoc. Consider merging or updating hb-257.
- Compute: 16-path SCED is ~16× the per-candidate LDPC cost. Current
  pipeline runs multi-pass SIC; SCED would replace or gate the deep-OSD
  pass where BP failed. Acceptable on Fast/Moderate tier hardware.

**License**: Academic arXiv preprints; no GPL. Reference Fortran/MATLAB
implementations on author pages would need clean-room treatment if used
directly; implement from the published equations.

**Draft bank entry**:
```
## hb-262 — Subcode Ensemble Decoding (SCED / aSCED)

Status: PENDING
Priority: 0.35
Source: arXiv:2501.11993 (SCED, Jan 2025); arXiv:2604.06889 (aSCED, Apr 2026);
         web-scan-2026-06-22.md

Premise: For candidates where single-path BP fails to converge, an ensemble
of BP decoders on subcodes (or affine subcodes) of the FT8 LDPC(174,91)
parity-check matrix can converge where the full-matrix decoder stalls.
Unlike AED (hb-257's automorphism approach), SCED requires no automorphism
group computation; subcodes are derived by random row-masking (documented in
arXiv:2501.11993, Algorithm 1). aSCED additionally uses affine (coset) shifts
for 0.2–0.3 dB over plain SCED.

Expected gain: 0.2–0.5 dB at equal decode time to OSD-depth=1 (per CCSDS
analogy). Measured on codes structurally similar to FT8 LDPC(174,91).

Architecture: ldpc.rs — after BP failure, before or instead of OSD. A
subcode-generator builds N masked H matrices at startup; the SCED ensemble
runs on BP-failure candidates only. N is a config knob (default 8).

Relationship to hb-257: SCED subsumes/supersedes the hb-257 "automorphism
permutations" approach for FT8 because: (a) FT8's automorphism group is
unknown; (b) SCED is strictly more general and measured better. Update
hb-257 with a reference here; measure SCED instead of hb-257's approach.

Graduation rule: Δ TPs ≥ +20 on raw_530_full with Δ FPs ≤ +50 at N=8 paths.
```

## Documented dead end

### hb-263 — Neural M-FSK demodulation inferior to max-log (DOCUMENTED-DEAD-END)

**Finding**: CNN and DNN demodulators for M-FSK (tested on JT65A's 65-FSK)
achieve 1.5–2 dB BELOW the theoretical noncoherent limit at moderate SNR, and
cannot beat max-log (dual-max) LLR computation. Pancetta already uses max-log,
which is the optimal practical approximation for noncoherent FSK. Neural
demodulation for 8-GFSK FT8 would be strictly worse.

**Sources**: arXiv:2502.19097 "Deep Learning for JT65A Demodulation" (Feb 2025);
arXiv:2502.16371 "Neural FSK Demodulation Benchmarks" (Feb 2025).

**Verdict**: Do not pursue. Neural demodulation for FT8 is a known-bad path.
Add to bank as DOCUMENTED-DEAD-END to close the line for future scans.

**Draft bank entry**:
```
## hb-263 — Neural M-FSK demodulation (DOCUMENTED-DEAD-END)

Status: DOCUMENTED-DEAD-END
Priority: 0.00
Source: arXiv:2502.19097; arXiv:2502.16371 (Feb 2025); web-scan-2026-06-22.md

Finding: CNN/DNN M-FSK demodulators for JT65A (65-FSK) measured 1.5–2 dB
BELOW theoretical noncoherent limit. Max-log (which pancetta uses) is the
near-optimal practical LLR approximation. Neural demodulation adds complexity
with strictly worse BER. Closed for FT8 8-GFSK by analogy: the gain gap
would be similar or larger (FSK order lower, coherence shorter). Do not pursue.
```

entry; monitor for a v3 paper if the MSHV author publishes.

**GRAND family** (previously hb-260 DEAD-END, confirmed): FT8 rate-0.52 with
83 redundancy bits is far outside GRAND's tractable guessing range. Confirmed
dead; no re-entry.
