## What this experiment did

Wired Phase 5 safety-driver kill switch into the TUI and coordinator.
Pressing Shift+Q in pancetta's terminal UI now:

1. Aborts any in-flight TX within ~50ms (PttGuard drop via the
   existing `abort_current_tx` Arc<AtomicBool> — same mechanism `h`
   already uses).
2. Disables autonomous mode at runtime via a new
   `autonomous_enabled_runtime: Arc<AtomicBool>` on
   `ApplicationCoordinator`. The autonomous decision loop reads this
   every slot before dispatching TX items. If false, the loop logs at
   WARN with `target: "operator.override"` and clears tx_items for
   that cycle. Internal state (decision engine, evaluator) is
   preserved so re-enabling resumes cleanly.
3. Stops the repeating-CQ loop and any active 12-second tune tone so
   no new TX gets armed in the next cycle.
4. Renders a red "STOPPED BY OPERATOR" banner across the top three
   rows of the TUI. Non-modal — other keys keep working so the
   operator can inspect band activity / waterfall / QSO state. Esc
   clears the banner.

All operator-override paths log at WARN with
`target: "operator.override"` so disengagements are greppable.

## Key bindings

- `Shift+Q` = emergency stop (new — hb-161).
- `q` (lowercase) = quit-confirm modal (unchanged).
- `h` = halt current TX only, autonomous keeps running (unchanged).
- `Esc` (when banner visible) = clear the banner.
- `a` = toggle autonomous (used to re-enable after a Q-press).

Help overlay (`?`) updated to list the new binding.

## Architecture notes

- New `TuiCommand::OperatorEmergencyStop` variant routed via the
  existing crossbeam command channel.
- New `pancetta-tui::App::stopped_by_operator: bool` field. Flipped
  locally on Shift+Q so the banner appears immediately, without
  waiting for the coordinator round-trip.
- The autonomous runtime gate seeds from `config.autonomous.enabled`
  at component start, so launching with `enabled = false` makes Q a
  no-op (idempotent safety-driver property).
- Re-enabling autonomous after a Q-press is an active operator
  decision (TUI `a`). The gate does NOT auto-restore. This is the
  documented safety-driver contract.
