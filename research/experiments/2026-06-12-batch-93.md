## What shipped

1. **Safety-recovery path (the Phase 5 blocker)**: after a Shift+Q
   emergency stop there was no way to resume autonomous operation
   without restarting pancetta — the `a` key only mutated TUI-local
   state, and the relay's own comment falsely claimed otherwise. Now:
   `TuiCommand::ToggleAutonomous` flips the same
   `autonomous_enabled_runtime` atomic the emergency stop clears
   (symmetric by construction; gate-seam unit test). End-to-end trace
   in the note: Shift+Q → TX aborted ≤50 ms + gate closed → `a` →
   gate reopens → autonomous resumes through its own decision gates.
   The toggle cannot itself start a transmission.
2. **Live `[AUTO]` panel**: structured `AutonomousStatusUpdate`
   (relay ANDs the runtime gate so the panel shows effective state;
   config-disabled reported honestly).
3. **TX-active badge**: `TxStatus` messages bracket all three TX arms
   via RAII guard (clears on abort/shutdown like PttGuard).

7 new tests; full workspace exit 0. No TX/PTT behavior changed —
observers and gate-flippers only.
