## Clean-room hygiene findings (from reader agents)

### Reader #1 (wsjtr sync) — caught 2 doc-vs-source discrepancies

- wsjtr's docs claim "depth=1 → 1.0×, depth=2 → 0.9×, depth=3 → 0.83×
  sync_min scaling" — NO such scaling in source; flat 1.2 threshold
- wsjtr's docs claim "(0.04s, 4 Hz) proximity dedup" — those are
  refinement grid step sizes, not dedup; no proximity dedup exists
- Spec flags both so implementer doesn't chase phantom mechanisms

### Reader #2 (wsjtr decode/OSD) — flagged paired dependency

- zsum OSD init **must ship WITH** the OSD distance-tracking fix
- Without distance-fix, zsum-init ships a known FP regression (the
  exact bug Bodiya documented in Feb 2026)
- Spec makes this dependency explicit

### Reader #3 (wsjtr A7/multipass) — flagged FP-amplification risk

- Cross-sequence A7: if seed callsigns are FPs, generates fake
  templates for next slot — amplifies FPs
- Mitigation: gate seed callsigns through hb-062 trust set + dmin/dmin2
  margin (1.3×)

### Reader #4 (ft8mon) — flagged possibly-verbatim line

- One function signature in `soft_decode_pairs` spec is a structural
  anchor that may be verbatim from ft8mon source
- Self-flagged for review before implementation

### Reader #5 (JTDX) — bonus finding outside scope

- JTDX `lqsothread` mode inserts 2 virtual candidates at `nfqso ± 5s`
  DT — force-emit family member worth a separate spec
- Cleaned up JTDX clone from /tmp after spec-writing (no contamination)
