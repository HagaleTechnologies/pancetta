## Root cause: corpus composition mismatch

hard-200 is curated from normal FT8 amateur traffic. Normal QSO operation is overwhelmingly:
- `CQ <call> <grid>` (StandardMessageType::Cq)
- `<call1> <call2> <grid|report|RRR|RR73|73>` (StandardMessageType::Reply etc.)

These are all `MessageType::Standard` in pancetta's parser, mapped to `MessageCategory::Standard` in FDR, which is treated as "easy" (no gating).

FreeText, Telemetry, DXpedition Fox/Hound, and contest-non-standard messages appear in special operating contexts. They are rare-to-absent in casual band recording.

**FDR's value, as the WSJT-X release notes describe, is for OPERATIONS where unusual types appear** — contests, DXpeditions, beacon networks. Pancetta's autonomous-station target (everyday FT8 amateur traffic) is exactly the regime where Standard dominates, and FDR has no leverage.

## Ship decision

**Default-OFF.** Operators can enable FDR (Level1 or Level2) when operating in special contexts. The module surface is complete:

```rust
use pancetta_qso::{FdrLevel, FdrFeatures, MessageCategory, fdr_should_reject};
// In a special operating context:
let level = FdrLevel::Level2;
let is_special_mode = true; // contest active
let category = translate(decoded_message.message.message_type);
let features = decoded_message.confidence_features.map(translate_features);
if fdr_should_reject(category, features.as_ref(), level, is_special_mode) {
    // Drop this candidate.
}
```

This is the operator's choice, not a project-level default.

**Coordinator wiring is deferred to a future session** (Session 4c). At default-OFF the wiring is a no-op, so its absence doesn't affect production. When an operator asks to enable FDR for contesting, Session 4c lands the wiring + a config knob.

1. **9 of N implementer-thread mechanisms now confirmed inapplicable on pancetta's corpus.** The pattern is robust: peer-decoder mechanisms designed for general-purpose FT8 receivers don't engage with pancetta's stack + corpus + measurement gates. The bootstrap-CI discipline is doing exactly what it's supposed to: surfacing mechanisms AND identifying which ones don't pay their way in pancetta's regime.

2. **The FDR infrastructure is valuable even without graduation.** ConfidenceFeatures (Sessions 1-3) is broadly useful: hb-103 content scoring, autonomous TX gating, research scorecard stratification. The decoder telemetry pays its way through other consumers regardless of FDR.

3. **The clean-room extraction process has high "find" rate but low "graduate" rate** at this point. Of 12+ implementer-thread mechanisms, only LLR whitening has graduated to default-ON. The others ship opt-in or get shelved. This is a known property of the pipeline; the mechanism inventory matters even when graduation rates are low because each "shelved-with-reason" finding eliminates a category of future investigation.

4. **The corpus diversity gap is the binding constraint.** To productively measure FDR, MTD staged-decoder, hb-244 soft combiner, hb-237 cross-sequence A7, etc., pancetta needs corpora that exercise:
   - Unusual message types (FreeText, Telemetry, DXpedition) for FDR
   - Live QSO continuity (operator-in-QSO partner replies) for hb-237
   - Repeat-heavy traffic (same caller heard 3-5x in 45 min) for hb-244 soft combiner
   - Freq drift (mobile, aurora, chirpy rigs) for Batch 50 freq tracker

   The autonomous loop has exhausted measurements achievable on existing corpora. Next-phase progress requires corpus engineering as the gating activity.
