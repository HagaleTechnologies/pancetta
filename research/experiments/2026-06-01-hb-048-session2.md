## What was built

### `pancetta-ft8::a7` module (943 LOC including tests)

**Types**
- `A7SlotParity` — `Even` / `Odd`, with `.opposite()`.
- `A7ExpectedCall` — `{ call, freq_hz, slot_parity, decoded_at,
  heard_with, my_call }`. Builder methods `with_heard_with`,
  `with_my_call`.
- `A7TemplateKind` — `StatusFromC` / `StatusToC` / `ReportFromC` /
  `ReportToC` / `CalledByCGrid` / `CqFromC` / etc.
- `A7Template` — `{ message_text, codeword[174], pinned[174], kind }`.

**Functions**
- `generate_templates(call) -> Vec<A7Template>` — builds up to 32
  templates per source decode (the CPU-bounded subset spec'd in §risk-3).
  Priority order: status messages with `my_call` → with `heard_with` →
  reports → grid → CQ → fallback.
- `cross_correlate(template, residual_llrs) -> A7Score` — computes
  `snr7 = sum(llr_i * (1 - 2*expected_bit_i)) / sqrt(N)`. Returns
  `{ snr7, bits_compared, raw_sum }`. Matches `par_compute_soft_llrs_db`
  sign convention (positive LLR = bit 0).
- `best_template_score(templates, llrs) -> Option<(idx, snr7, snr7b)>` —
  returns the best-scoring template index, its snr7, and the
  best/second-best ratio (the WSJT-X AP-FP filter).
- `dedup_against_previous(new, prev, freq_window_hz)` — the WSJT-X
  `f0 = -98.0` analog: drop entries already seen one slot ago at the
  same audio frequency.

**Helpers**
- `symbols_to_codeword_bits` — Gray-decode the 79-symbol output of the
  encoder back to a 174-bit codeword. (The encoder's `payload_to_ldpc`
  is private; this is the cleanest workaround without modifying the
  encoder.)
- `build_standard_pinned_mask` — sets PINNED at bits 0-27, 28-55, 74-76
  (callsign A, callsign B, i3), with an optional pin-payload flag for
  the 18 grid/report bits (status messages pin them; reports don't).

### Synthetic-injection micro-test

`pancetta-research/examples/hb048_a7_synthetic_injection.rs`.

**Setup**
- Source decode: `K1ABC` at 1200 Hz, even slot, `heard_with=W1AW`,
  `my_call=W1AW`.
- Truth follow-up: `K1ABC W1AW RR73`.
- Templates generated: **22** (cap is 32; floor for sanity is 8).
- Matching template found at **index 2** (kind=`StatusFromC`).

**Noise sweep** (signal_mag=5.0 LLR units, 5 trials per point):

| noise_std | lin SNR | snr7 | snr7b | best-correct |
|-----------|---------|------|-------|--------------|
| 0.5       | +20.0dB | 66.06 | 1.85 | YES (5/5) |
| 1.0       | +14.0dB | 65.93 | 1.85 | YES (5/5) |
| 1.5       | +10.5dB | 65.75 | 1.88 | YES (5/5) |
| 2.0       | +8.0dB  | 66.12 | 1.91 | YES (5/5) |
| 3.0       | +4.4dB  | **65.38** | **1.83** | **YES (5/5)** |
| 5.0       | +0.0dB  | 63.53 | 1.97 | YES (5/5) |
| 8.0       | -4.1dB  | 63.55 | 1.70 | YES (5/5) |
| 12.0      | -7.6dB  | 74.96 | 1.63 | YES (5/5) |

(snr7 going *up* at noise=12 is a small-sample artifact — random
alignment between the noise realization and the true codeword
boosts the matched-filter integral. The thing that monotonically
degrades is **snr7b**, which is the right metric for FP guard.)

**Headline**: at the mid-band operating point (noise_std=3.0, lin SNR
+4.4 dB) → snr7=65.38, snr7b=1.83, best-template correctly identified
in 5/5 trials.

## Open observations for Session 3

1. **snr7b sits very close to 1.8 by design** — the bank has multiple
   structurally-similar templates (RRR vs RR73 vs 73, all with the same
   callsign pair). The second-best is the template that differs from
   the winner only in the payload bits. This is a structural ceiling on
   snr7b that Session-3 should map: at template bank N=22, snr7b≈1.85
   is the expected best case. Session 3's threshold sweep should probe
   whether 1.8 is the right gate or whether 1.5 (with extra
   precision-guarding via the strict FP filter) recovers more.

2. **The snr7 absolute scale is unit-dependent.** Our snr7 here uses
   `signal_mag=5.0` LLR-units (a typical decoder LLR magnitude). In
   production, `par_compute_soft_llrs_db` produces LLRs with empirical
   variance set by `llr_target_variance` (config). The threshold of 6.0
   from WSJT-X is *not directly transferable* — Session 3's sweep MUST
   include a wide bracket around it.

3. **Template count of 22 is below the WSJT-X reference of 206** by
   design (spec §risk-3: per-decode cap of 32). For Session 3, instrument
   per-WAV: how often does the per-decode template bank size hit the
   cap? If usually ≤22, then increasing the cap costs nothing; if often
   at 32+, raising the cap is a free recall lever.

4. **Round-trip codeword reconstruction works correctly.** The
   `symbols_to_codeword_bits` helper (Gray-decoding the encoder's 79-symbol
   output back to 174 bits) round-trips cleanly — verified by the
   `test_codeword_reconstruction_round_trip` test (snr7 > 100 against
   clean LLRs). This is the path Session 3 will use for the production
   template-codeword generator.
