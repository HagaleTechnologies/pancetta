## Major findings

### F3: hb-062 FP filter catches 100% of pure-noise FPs

Batch 28 D found 71% of pure-noise windows emit ≥1 CRC-passing decode
(254 FPs across 200 noise windows at varied σ). Batch 29 F examined
the **callsigns** in those FPs:

- 502 callsigns extracted from 261 FP messages
- **0/502 (0.00%) appear in hard-200 truth callsign set (2512 unique)**

The hb-062 callsign-continuity FP filter, which rejects messages whose
callsigns don't appear in the operator's ADIF / cqdx-spotted / rolling-
window trust set, would catch **100%** of pure-noise FPs based on
hard-200 truth alone. In production with a larger trust set (the
operator's own ADIF + live cqdx spots), coverage would be even better.

The 71% raw FP rate is essentially **operationally meaningless** —
production filtering eliminates it cleanly. Pancetta is recall-first +
filter-precise by design, and the design is working.

### H1: hb-058 /R-filter PROCEED — 100% FP rate, 0 cost

On 20 top-hard-200 WAVs:

- 723 total pancetta decodes, 512 TPs (overall FP rate 29.18%)
- **62 /R-containing emissions, 0 TPs (100% FP rate)**
- 0 /R truths in hard-200 jt9 baselines for these 20 WAVs

A trivial filter dropping any decode whose callsign suffix ends in
`/R`, `/R1`, or `/R2` would eliminate 62 FPs (~3/WAV) with zero
recall cost on this corpus. This is the simplest production
quick-win available — ~5 LOC in the existing FP filter, no API change.

Operational impact: ~29% reduction in FP load (62 / (723-512) ≈ 29.4%).

### J: hb-100 capture-effect threshold map

Synthetic interferer pairs at controlled (Δfreq, B-amplitude):

```
Δfreq=6.25 Hz, equal amplitude:   neither A nor B decodes in combined
Δfreq=12.5 Hz, equal amplitude:   neither A nor B decodes in combined
Δfreq=25 Hz+,  equal amplitude:   both A and B cleanly decoded

Δfreq=6.25 Hz, 6 dB asymmetric:   A always; B 2/20
Δfreq=12.5+ Hz, asymmetric:       both cleanly decoded
```

**Capture-effect boundary on pancetta = ~25 Hz for equal amplitudes**
(4 freq bins at 6.25 Hz/bin spacing). For 6 dB asymmetric pairs, A is
always recoverable; B is blocked only at 6.25 Hz (1 bin).

This is the synthetic baseline for hb-079/080/086 joint-pair-retry
work. PROCEED-INFO: hb-100 data is now available; whoever picks up
joint-pair-retry next can validate their mechanism against this
synthetic corpus before going to operator-captured data.

### F1/F2: FP message-type and prefix distribution

- 97.3% of pure-noise FPs are "grid_or_other" (two-callsign + grid
  exchange pattern), only 0.8% are CQ
- Callsign prefix-letter distribution is uniform-ish across 25
  letters (4-5.8% each for top 15)
- (freq, dt) distribution: spans full 16-3991 Hz freq range and
  0-3.84 s dt range — uniform across the slot

The structure of CRC-passing-on-noise FPs: random-looking codewords
that happen to encode callsign-1 + callsign-2 + grid more often than
CQ messages, because CQ messages have a more constrained free-text
field that's harder to land randomly.

### I: hb-097 synthetic α* validation

Clean reference signal + Gaussian noise, sweep α ∈ [0.7, 1.3], find
α* minimizing residual energy. At σ ∈ {0.001, 0.01, 0.05}, α* = 1.0
exactly in every realization (50 per σ).

This validates the line-search math but does NOT test the production
hb-097 hypothesis (that pancetta's internal rotor noise produces
α-drift in `subtract_decode_coherent`). The production question
remains open until internal subtract-path instrumentation lands.

Honest framing: MECHANISM-VALIDATED, not PROCEED or SHELVE. The bank
entry's "PROCEED if median |α-1| ≥ 0.05" requires production data.
