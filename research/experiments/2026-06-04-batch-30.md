## Research diagnostics

### Diagnostic M — hb-103 FP-filter AUC characterization

Operating curve of `CallsignContinuityFilter` at varying trust-set
sizes on 787 decodes (top-20 hard-200 + 50 noise windows):

| Trust set | TP recall | FP reduction | Lift (TP/FP) |
|---|---|---|---|
| 0 (no filter) | 100.0% | 0.0% | 1.86 |
| 5% (137 calls) | 15.8% | 95.6% | 6.75 |
| 10% (274) | 23.8% | 91.3% | 5.08 |
| 25% (684) | 55.3% | 81.5% | 5.55 |
| 50% (1368) | 82.8% | 76.0% | 6.42 |
| **100% (2736)** | **100.0%** | **67.3%** | **5.69** |

**Key finding**: at 100% trust (full hard-200 callsign coverage), FP
reduction caps at 67.3%. The remaining 33% of FPs contain trusted
callsigns attached to wrong messages — pure callsign-continuity can't
filter them. **That 33% gap is hb-103's territory** (continuous
trust-score using message-level features beyond callsign membership).

Quantifies the headroom hb-103 would address: ~90 surviving FPs in this
slice. Not enormous, but precision-only — every one eliminated is
zero-cost.

### Diagnostic N — synthetic SNR-recall curve

Pancetta-ft8 decode rate on synthetic clean signal + Gaussian noise,
SNR measured in 2500 Hz reference BW:

| SNR (dB) | success / total | rate |
|---|---|---|
| -10 | 50/50 | 100.0% |
| -14 | 50/50 | 100.0% |
| -18 | 47/50 | 94.0% |
| **-21** | **0/50** | **0.0%** |
| -25 | 0/50 | 0.0% |
| -28 | 0/50 | 0.0% |

**Sensitivity cliff at -18 to -21 dB.** pancetta's synthetic
performance is ~3 dB worse than WSJT-X reference (which is the FT8
sensitivity benchmark). Production config (`Ft8Config::default()`)
sits ~3 dB above the WSJT-X cliff.

This characterizes pancetta's intrinsic sensitivity. Real-world recall
gaps vs WSJT-X have two sources: this synthetic-floor gap (~3 dB
sensitivity deficit) + corpus/channel effects (interference, fade,
Doppler). The 5-10%-of-WSJT-X recall claim from CLAUDE.md is
consistent with sensitivity + corpus contributions stacking.

### Diagnostic O — pancetta recall by SNR bucket on lid_of_band

Top-20 lid_of_band (weakest-truth WAVs from Batch 29 G's shipped
manifest):

| SNR bucket | truths | recovered | recall |
|---|---|---|---|
| ≥ -10 dB | 438 | 331 | **75.6%** |
| -15..-10 | 172 | 85 | 49.4% |
| -19..-15 | 96 | 43 | 44.8% |
| -22..-19 | 65 | 21 | 32.3% |
| ≤ -22 | 99 | 35 | 35.4% |
| TOTAL | 870 | 515 | 59.2% |

**Key finding**: only 75.6% recall even at ≥ -10 dB. The 24% miss on
strong signals isn't a sensitivity problem — it's coverage. Most
likely sources:
- Capture-effect on co-located signals (Batch 29 J: ≤25 Hz Δfreq blocks both)
- Message-type variants pancetta doesn't fully decode (Field Day, RTTY
  Roundup nuances)
- Sync failures on slot-misaligned recordings

Recall has a hard cliff transition at -15 dB (75.6% → 49.4%) but then
plateaus around 30-45% from -19 dB down. The "below -22 dB" recall
(35.4%) is actually HIGHER than -22..-19 (32.3%) — likely a jt9 SNR
saturation effect (jt9 floors negative SNRs around -25, so signals
that hit the floor may actually be variable).

Future recall hypotheses should be SNR-bucket-stratified:
- "Recall on strong signals" → target capture-effect, message types
- "Recall on cliff signals" → target sensitivity (matched filter,
  coherent integration)

### Diagnostic P — coherent vs non-coherent integration

**Wrong-scoped.** Tested 10-symbol single-tone coherent (sum I+jQ
across all symbols, take magnitude) vs "non-coherent" (per-symbol
envelope, summed). Result: -0.01 to -0.25 dB across SNR levels.

The "non-coherent" branch is actually per-symbol-coherent — it already
contains symbol-level coherent integration. The proper test would
compare full-frame phase-coherent integration vs per-symbol-only.

Honest framing: this diagnostic doesn't address hb-090's mechanism.
hb-090 requires coherent integration across 79 symbols at KNOWN tone
positions (AP-derived), which needs structural pipeline changes
(complex-spectrogram retention + per-symbol phase model). My
single-tone synthetic test can't capture that.

Verdict: SKIP — hb-090 mechanism question remains open and requires
proper IQ-domain instrumentation.
