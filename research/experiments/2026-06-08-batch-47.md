## Key strategic findings

### 1. WSJT-X mainline reading caught wsjtr translation artifacts

Agent B (mainline) found at least 3 substantive mechanisms wsjtr's docs MISSED or PARAPHRASED AWAY:

- **TWO parallel sync baselines** (red ±10-col tight + red2 ±62-col WIDE), each 40th-percentile normalized — the WIDE-LAG baseline is what catches slot-edge negative-dt signals. wsjtr only described the tight-lag version. **This re-evaluates hb-242 design entirely.**
- **FOUR parallel LLR variants A/B/C/D** in ft8b — superset of ft8mon's `soft_decode_pairs` (hb-223)
- **`npre2` OSD preprocessing rule** — hash-table-based complementary-bit-pair search via `boxit91`/`fetchit91` at ndeep≥3. **wsjtr does NOT implement this.**

**Lesson**: clean-room reading the original source (not just downstream ports) catches translation gaps. The Reader → Spec → Implementer firewall doesn't just protect license; it also ensures we have the right detail level.

### 2. JS8Call-Improved has 4 mechanisms that exist nowhere else

Agent F's biggest find: JS8Call-Improved (Chris AC9KH + Jordan KN4CRD) eliminated all Fortran, ported decoder to C++, and introduced 4 genuinely novel mechanisms:

1. **LDPC feedback refinement** (~150 LOC) — meta-loop using prior hard-decision codeword
2. **Soft combiner** (~250 LOC) — **HIGHEST-LEVERAGE for hb-218 weak-signal**; LRU cache + Hamming-≤4 fuzzy match + additive LLR combination across repeated receptions = time-diversity boost
3. **LLR whitening** (~150 LOC) — per-tone × per-symbol noise normalization
4. **Per-candidate frequency tracker** (~150 LOC) — adaptive PLL/Kalman using Costas residuals

**No other FT8 decoder has these.** Pancetta could be first to ship.

### 3. Parameter values in docs are often wrong

Surfaced corrections:
- **subtract_ramp** = 17.6 ms (NOT 3.5 ms as some docs claimed)
- **napwid** = 5 Hz HF / 15 Hz VHF / 50 Hz UHF (NOT 75 Hz as initial brief said)
- **wsjtr docs claimed phantom sync_min depth-scaling** that doesn't exist in source
- **wsjtr docs claimed phantom proximity dedup** that doesn't exist in source

Always validate parameter values against actual source.

### 4. License vetting closed several apparent opportunities

Agent G surfaced license traps:
- **kenrduffy MIT GRAND** repos are actually "Non-Commercial Academic Research Use License 021722" — looks like MIT but isn't OSS
- **CrossMPT** is "non-commercial research only"
- **Lugosch Neural Min-Sum** is CRAPL non-commercial
- Only **ECCT** (yoniLc) is actually OSI MIT among neural-LDPC implementations
- **kholia/pico_ft8_xcvr** is genuinely MIT — usable

### 5. QSO-partner-zone bypass principle

JTDX has this design pattern in 3 mechanisms (lft8subpass nweak=2, windowed-sync-guard Phase 5 bypass, relaxed-sync-near-partner): all gate on `dfqso < 2 Hz`. "Give the operator's partner the benefit of the doubt." Pancetta should be internally consistent across mechanisms.

## Process learnings

1. **Wide-net sweep paid off**. 5 projects beyond Batch 46's initial 3 (wsjtr/ft8mon/JTDX). 11+ new bank candidates surfaced.

2. **Reader → Spec → Implementer firewall scaled to 9 parallel agents**. 4. **Some "documented improvements" are infrastructure for adjacent protocols**. SDRangel libft8's "2024 changes" are FT-chirp infrastructure with zero FT8 delta. Validate the consumer before assuming the mechanism helps.

5. **License vetting is its own discipline**. "MIT-named-but-non-commercial" repos exist; pancetta would have contaminated itself by assuming MIT == MIT.
