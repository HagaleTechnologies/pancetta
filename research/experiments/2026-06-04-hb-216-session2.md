## Motivation

S1 (1c40954) shipped the standalone hardware-tier probe + classifier
inside `pancetta-ft8::tier_probe`. The probe ran via the
`pancetta-research::tier_probe` CLI example and emitted recommendations,
but nothing in the production coordinator consumed them — operators
still had to set `PANCETTA_SCOPED_FAST_PATH=1` by hand on
constrained hardware.

S2 closes that loop. On boot, pancetta probes itself (background,
non-blocking), persists the classification, and applies the per-tier
runtime preset automatically.

## What shipped

### New module: `pancetta/src/coordinator/tier.rs`

- `Override` enum (`ForceOn`, `ForceOff`, `None`) parsed from the
  three-state env var `PANCETTA_SCOPED_FAST_PATH`.
- `TierCache` struct, schema_version 1, JSON at
  `~/.pancetta/tier_cache.json`. Atomic write (temp + rename) on save;
  tolerant load (missing / malformed / schema mismatch → `None` and
  re-probe).
- `current_hardware_key() -> (cpu_model, core_count)` — platform-probed
  via `sysctl` (macOS), `/proc/cpuinfo` (Linux), `wmic` (Windows), with
  `std::env::consts::ARCH` fallback. `num_cpus::get()` for the count.
- `apply_tier(tier, override, &AtomicBool, &RwLock<Ft8Config>)` — the
  pure-policy function. Returns a human-readable summary line. Honors
  override completely (override skips both the atomic flip *and* the
  Slow-tier preset rewrite).
- `initialize(Arc<RwLock<Ft8Config>>) -> Arc<AtomicBool>` — the
  coordinator-startup entry point. Seeds the atomic from env, loads
  cache, applies if fresh, spawns the probe worker if stale/missing.
- Probe worker (`spawn_probe_worker`) — `tokio::task::spawn_blocking`
  runs `probe_hardware_tier(10)` (~2s on Fast, ~30s on Slow), writes
  cache, applies tier via `apply_tier`. Failures log at `warn!` and
  leave defaults intact — never panic.

### Coordinator state additions: `pancetta/src/coordinator/mod.rs`

Two new `ApplicationCoordinator` fields:

```rust
scoped_fast_path: Arc<AtomicBool>,
ft8_config: Arc<RwLock<Ft8Config>>,
```

`new()` constructs `ft8_config = Arc::new(RwLock::new(Ft8Config::default()))`
and calls `tier::initialize(ft8_config.clone()).await` to seed the
atomic.

### Hot-loop integration: `pancetta/src/coordinator/ft8.rs`

Two changes inside `start_ft8_pipeline`:

1. **Atomic replaces env-var read.** The check that previously was
   `std::env::var("PANCETTA_SCOPED_FAST_PATH").as_deref() == Ok("1")`
   is now `scoped_fast_path.load(Ordering::Relaxed)`. The cloned
   `Arc<AtomicBool>` is captured into the `spawn_blocking` closure.

2. **Decoder rebuild on shared-config change.** Before each window's
   decoder use, the loop runs a non-blocking `try_read()` on the shared
   `Ft8Config`. If `(max_decode_passes, osd_depth)` differs from the
   tuple the current decoder was built with, the decoder is rebuilt
   from the new config. In practice this happens **at most once** per
   process lifetime: the probe lands once, writes the Slow preset, the
   decoder rebuilds on the next window.

The local `let ft8_config = Ft8Config::default()` line goes away; the
initial decoder is built from a snapshot of the shared config taken
before the FT8 thread spawns.

## Override matrix (final)

| `PANCETTA_SCOPED_FAST_PATH` | Probe tier | `scoped_fast_path` atomic | Slow preset applied? |
|---|---|---|---|
| (unset) | Fast      | `false` | no  |
| (unset) | Moderate  | `true`  | no  |
| (unset) | Slow      | `true`  | yes |
| `"1"`   | (any)     | `true`  | no  |
| `"0"`   | (any)     | `false` | no  |

`=1` and `=0` are operator overrides; they short-circuit *both* the
atomic flip and the Slow-preset rewrite. Rationale: when the operator
forces the knob, we don't quietly also rewrite `max_decode_passes` /
`osd_depth` on their behalf. They asked for one specific change; that's
what they get.

## Cache schema (concrete)

```json
{
  "schema_version": 1,
  "cpu_model": "Apple M4",
  "core_count": 10,
  "pancetta_version": "0.1.0",
  "tier": "fast",
  "p50_ms": 210,
  "p95_ms": 213,
  "p99_ms": 213,
  "probed_at": "2026-06-04T12:34:56Z"
}
```

Cache key match requires all three of `(cpu_model, core_count,
pancetta_version)` to equal the current host. Mismatch on any → re-probe.
`schema_version != 1` → re-probe (forward-compat hook for future S3+).

## What S2 deliberately defers

- **Cross-hardware threshold tuning**. Fast/Moderate/Slow boundaries
  remain the S1 synthetic-baseline values. Refinement waits on
  measured data from MiniPC / ARMv7 deployments.
- **TUI surfacing of the classification**. Logs only — operators can
  `grep tier` in the log stream.
- **`pancetta-config` schema field** for tier override. Env var
  suffices for the foreseeable future; adding a config field doubles
  the surface area without a real ergonomics win.
- **Cache-write-during-probe-failure metric**. The probe-failure path
  just logs and bails; no failure counter.

## Lessons / notes

1. **`Arc<AtomicBool>` is cleaner than `std::env::set_var` from a
   background thread.** Rust 2024+ is moving toward making `set_var`
   explicitly `unsafe` (concurrent reads of the process env are racy).
   Replacing the env-var read in the hot loop with an atomic load
   sidesteps that landmine and gives us a cheap, lock-free wire.
2. **`tokio::sync::RwLock::try_read()` makes per-iteration config
   re-reads zero-cost when uncontended.** The only place writes happen
   is the one-shot probe-worker call to `apply_tier`. Reads happen
   per-window in the hot loop. Lock contention is near-zero in
   practice; even on the slot the probe lands, `try_read` either
   succeeds or the loop skips the check and picks it up next slot.
3. **Defensive `max_decode_passes = 1` in the Slow preset.** The
   current `Ft8Config::default()` already sets `max_decode_passes = 1`
   (hb-031 baseline), so the explicit assignment is a no-op today. It
   stays in the preset to insure against a future default-bump
   regressing the Slow tier silently.
4. **The hb-091 + hb-216 pair is now operationally complete.**
   hb-091 shipped the *capability* (scoped fast-path, env-var gate).
   hb-216 S1 shipped the *measurement* (probe + classifier).
   hb-216 S2 shipped the *decision wiring* (probe → atomic → hot loop +
   probe → config → decoder rebuild). A fresh-install MiniPC operator
   now goes from boot to optimal decoder config without ever touching
   an env var.
