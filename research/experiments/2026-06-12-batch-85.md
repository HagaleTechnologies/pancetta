## FFI fixes

Upstream ft8_lib's `ftx_decode_candidate` never writes `status.freq` /
`status.time` (only ldpc_errors + CRCs) — so every truth file carried
`freq_hz: 0.0`. Additionally `decode_window_ft8lib` fed the FFI tuple's
TIME element into the `snr` slot of `from_ft8lib` (truth snr_db values
were actually time offsets).

Fixed: freq/time now derived from the candidate exactly as upstream
`demo/decode_ft8.c` does; time routed to `time_offset`; `snr_db = 0.0`
recorded honestly (this path reports no SNR). Labeler gained
`PANCETTA_B71_FORCE=1` + `time_sec` field. Full regeneration: 25,105
files, 209s, 69,969 decodes — which exactly matches the Batch 76
per-day density table (resolving the apparent "14,951" discrepancy in
the old session notes: that was Batch 71's incremental count, not the
total).

## hb-104 premise diagnostic

Question (data-first, before any implementation): what fraction of
pancetta's misses are interferer-dominated — i.e., within capture-effect
range of a signal pancetta DID decode in the same slot?

Method: Batch 66 5/30 scan (2,066 slots of pancetta decodes with
freq/dt) vs refreshed ft8_lib truth (now with freq/time), time gate ±2s.

| Nearest decoded signal at Δf | misses | share |
|---|---:|---:|
| **< 6.25 Hz (one tone spacing)** | **821** | **46.5%** |
| 6.25–25 Hz | 35 | 2.0% |
| 25–100 Hz | 624 | 35.3% |
| ≥ 100 Hz / no time overlap | 286 | 16.2% |

(39,668 truth decodes, 1,766 misses = 4.5% miss rate; zero misses in
empty slots.)

**Premise CONFIRMED and sharper than hb-104 predicted**: the
interferer-dominated population isn't just "within 25 Hz" — it's
overwhelmingly co-channel (within ONE tone spacing), exactly the
capture-effect regime hb-100 mapped (≈25 Hz boundary at equal
amplitude, weaker signal blocked at Δf=6.25 Hz). This is the largest
single remaining recall opportunity: recovering a third of the <6.25 Hz
bucket ≈ +270 TPs on this corpus, vs +42 max from the entire knob lever
ladder.

Caveat: the scan is pre-Batch-72-config (richer decode set); under
today's defaults the decoded-signal set is ~2% smaller, so the overlap
shares shift only marginally.
