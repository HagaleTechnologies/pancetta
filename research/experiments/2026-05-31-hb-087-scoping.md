## Diagnostic result — kill-switch passed

Run: `cargo run --release -p pancetta-research --example hb087_callsign_priors_feasibility`

Method: for every missed truth in the top-20 worst hard-200 WAVs
(per `main.json` `per_wav_top_failures`), extract callsign tokens via
the same regex pancetta-qso uses, check membership in three prior sets:

- operator-callsign set ({K5ARH});
- recent-window decodes (this WAV's pancetta decodes, callsign-extracted);
  stands in for production's rolling 15-30 min window;
- bundled-common-active list (~100 hand-picked very-active stations);
  stands in for cqdx-spotted callsigns at runtime.

Aggregate over top-20 hard-200 (refreshed corpus, multipass N=3 + V1 ON):

| metric | value |
|---|---:|
| total truths | 1245 |
| recovered (pancetta production) | 558 |
| missed | 647 |
| missed with extractable callsign | 647 |
| covered by operator (K5ARH) | 0 (0.0%) |
| covered by recent-window | 150 (23.2%) |
| covered by bundled-common | 5 (0.8%) |
| **covered by ANY source** | **153 (23.6%)** |

PROCEED gate: ≥20% of missed truths have a callsign in the prior union.
**Headline = 23.6% — PROCEED.**

## Source-by-source interpretation

- **Operator (0.0%)**: expected on this corpus. The hard-200 WAVs aren't
  K5ARH's logs; nobody on these WAVs is calling K5ARH or being called by
  K5ARH. In production (field eval) the operator-call prior matters
  only when the operator is actively transmitting or in QSO, which is a
  different regime from passive hunting.
- **Recent-window (23.2%)**: dominant signal. This source mirrors
  pancetta-qso::callsign_continuity::CallsignContinuityFilter's rolling
  window. The same stations that produce decodes pancetta CAN see (the
  recovered 558) reappear in missed truths the production decoder
  misses elsewhere in the same band. Multi-decode stations form the
  natural prior universe.
- **Bundled-common (0.8%)**: hand-picked list of ~100 active /
  DXpedition stations. Small because the bundled list is small AND
  because these specific top-20 hard-200 WAVs don't carry DXpedition
  activity. Production's cqdx-spotted prior will be richer
  (band-filtered, time-filtered, larger universe) and more covering;
  treat 0.8% as a floor for the cqdx prior's contribution.

The **23.6% figure is a conservative lower bound** because:
- production rolling window is longer than per-WAV (15-30 min vs ~15s
  here);
- cqdx-spotted is broader than the bundled-100 stand-in;
- ADIF history (operator's QSO log) adds another source not measured
  in the diagnostic.

## Doctrine note — paired decodability test (gated at session 2/3)

The V3 SHELVE established: geometric proximity diagnostics need a
decodability sub-test before earning PROCEED. hb-087's coverage test is
NOT purely geometric (it measures callsign presence in priors, which is
a necessary mechanism input), but it's still not a *decodability* test.
Session 2 → 3 must gate on a per-truth decodability micro-test:

1. Pick 10 missed truths from the top-3 worst WAVs whose callsign IS in
   the prior set.
2. For each: extract residual LLRs at the truth's known (freq, time)
   coordinates from the production residual.
3. Inject the prior callsign at the appropriate AP position (AP1 if
   truth's called == prior, AP2 if truth's caller == prior).
4. Run `decode_soft` + CRC + plausibility.
5. Confirm ≥3 of 10 pass.

If <3 of 10 pass, the prior injection alone doesn't rescue marginal
residuals; SHELVE without implementing the full pass. This is the V3
doctrine refinement applied prophylactically.
