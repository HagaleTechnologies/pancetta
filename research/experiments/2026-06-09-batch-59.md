## Telemetry semantics, now complete

| Field | Source | Populated when |
|---|---|---|
| `bp_iterations_used` | `belief_propagation_with_features` convergence point | BP ran (always) |
| `min_llr_magnitude` | `min_abs_llr` over final 174-bit codeword | BP ran (always) |
| `osd_depth_used` | `OsdDecoder::decode_with_features` return | OSD invoked AND succeeded |
| `nharderrs` | `OsdDecoder::decode_with_features` return | OSD invoked AND succeeded |

When BP converges directly without OSD: `osd_depth_used = None, nharderrs = None`. When BP fails and OSD succeeds: all four fields populated. When BP fails AND OSD fails: the decode path returns `Ok(None)` (no `DecodedMessage` emitted at all).

The FDR-relevant convention now spans both stages:
- **High confidence**: low `bp_iterations_used`, high `min_llr_magnitude`, `osd_depth_used = None` (BP-direct path)
- **Lower confidence**: high `bp_iterations_used`, low `min_llr_magnitude`, high `osd_depth_used` (OSD had to work hard)

Per-message-type thresholds (Session 4) will encode these heuristics.
