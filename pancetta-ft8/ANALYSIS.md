# FT8 Encoder/Decoder: Honest Assessment

_Written 2026-03-01. Updated 2026-03-02 after completing cross-implementation validation._
_Based on thorough reading of all source files, tests, and comparison against
ft8_lib/WSJT-X reference implementation._

---

## Executive Summary

**The encoder is genuinely correct.** It produces bit-exact output matching
ft8_lib for payloads, LDPC codewords, CRC-14, Gray code mapping, and all 79
symbols. This is verified by 15 WSJT-X compatibility tests and 10 ft8_lib
cross-validation tests.

**The decoder works end-to-end.** All 6 critical bugs from the original
assessment were fixed. The message parser was rewritten with correct mixed-radix
callsign decoding, grid unpacking, and free text parsing. Full round-trip
(encode → modulate → decode) is verified for all standard FT8 message types.

**The modulator is clean.** Pure 8-CPFSK with rectangular pulse shape, continuous
phase accumulation, and 0.95 peak normalization. ft8_lib can successfully decode
audio generated by our modulator.

**Cross-implementation validation complete.** Our audio is decoded by ft8_lib.
ft8_lib's GFSK audio is processed by our decoder. Encoder output is bit-exact
for all standard message types.

**200 tests pass, 0 failures.**

## Part 2: What Remains

### Known Limitations

| Limitation | Impact |
|------------|--------|
| Free text encoding differs from ft8_lib | Both produce valid payloads, different binary representation |
| /R and /P suffixes are indistinguishable | Protocol-level: both encode as ip=1 |
| No i3=2 (EU VHF contest) support | Contest exchanges not decoded |
| No i3=3 (ARRL RTTY Roundup) support | Contest exchanges not decoded |
| No i3=4 (nonstandard callsign) support | Unusual callsigns not decoded |
| Off-air WAV decode rate is low | 3/9 files (ft8_lib also fails on these files) |
| Decoder not optimized for speed | Works but not benchmarked against real-time |

### Recommended Next Steps

1. **Off-air decode improvement**: Investigate why 6/9 WAV files fail (for both our decoder and ft8_lib)
2. **Performance benchmarking**: Measure decode time vs real-time requirement
3. **GFSK modulation**: Add Gaussian pulse shaping for better spectral efficiency
