## Task 1: LoTW Upload Response Parser

**File:** `pancetta-dx/src/lotw.rs`

**Problem:** `parse_upload_response()` (line 577) always returns `accepted: None, rejected: None`. The method receives the HTML response body from LoTW but only parses the status string and (incorrectly) the processed count. The accepted/rejected fields are left as `None`.

**Approach:** LoTW's upload response page includes lines like:
```
Your file has been queued for processing.
3 QSO records were uploaded.
2 QSO records were accepted.
1 QSO record was rejected.
```

Parse these with simple string scanning (no HTML parser crate needed — the counts appear in plain text).

### Steps

- [ ] **1.1** Replace the body of `parse_upload_response` in `pancetta-dx/src/lotw.rs` (lines 577-612):

```rust
/// Parse upload response HTML
fn parse_upload_response(&self, html: &str) -> Result<LotwUploadResponse> {
    let status = if html.contains("Upload successful")
        || html.contains("File uploaded")
        || html.contains("queued for processing")
    {
        "success".to_string()
    } else if html.contains("Error") || html.contains("Failed") {
        "error".to_string()
    } else {
        "unknown".to_string()
    };

    let processed = Self::extract_count(html, "uploaded");
    let accepted = Self::extract_count(html, "accepted");
    let rejected = Self::extract_count(html, "rejected");

    let errors = if status == "error" {
        // Try to extract error lines
        let error_lines: Vec<String> = html
            .lines()
            .filter(|line| {
                let lower = line.to_lowercase();
                lower.contains("error") || lower.contains("rejected")
            })
            .map(|line| line.trim().to_string())
            .collect();
        if error_lines.is_empty() { None } else { Some(error_lines) }
    } else {
        None
    };

    Ok(LotwUploadResponse {
        status,
        processed,
        accepted,
        rejected,
        errors,
    })
}

/// Extract a numeric count preceding a keyword from HTML text.
/// Matches patterns like "3 QSO records were accepted" or "1 QSO record was rejected".
fn extract_count(html: &str, keyword: &str) -> Option<u32> {
    for line in html.lines() {
        let lower = line.to_lowercase();
        if lower.contains(keyword) {
            // Walk backwards from the keyword to find the number
            let words: Vec<&str> = line.split_whitespace().collect();
            for (i, word) in words.iter().enumerate() {
                if word.to_lowercase().contains(keyword) {
                    // Look at preceding words for a number
                    for j in (0..i).rev() {
                        if let Ok(num) = words[j].parse::<u32>() {
                            return Some(num);
                        }
                    }
                }
            }
        }
    }
    None
}
```

```rust
/// Render help panel as an overlay
fn render_help_modal(f: &mut Frame, area: Rect) {
    let bindings = vec![
        ("Ctrl+Q / Esc", "Quit"),
        ("F1 / ?", "Toggle this help"),
        ("Tab / Shift+Tab", "Next / previous panel"),
        ("1-4", "Jump to panel"),
        ("Up / k", "Scroll up"),
        ("Down / j", "Scroll down"),
        ("PgUp / PgDn", "Scroll page"),
        ("Left / Right", "Band down / up"),
        ("t", "Toggle theme"),
        ("m", "Toggle monitoring"),
        ("a", "Toggle autonomous mode"),
        ("p", "Pause / resume autonomous"),
        ("D", "Audio device selection"),
        ("Ctrl+C", "Clear messages"),
        ("Enter", "Call selected station"),
    ];

    let modal_width = 50u16.min(area.width.saturating_sub(4));
    let modal_height = (bindings.len() as u16 + 4).min(area.height.saturating_sub(2));

    let modal_area = Rect {
        x: (area.width.saturating_sub(modal_width)) / 2,
        y: (area.height.saturating_sub(modal_height)) / 2,
        width: modal_width,
        height: modal_height,
    };

    f.render_widget(ratatui::widgets::Clear, modal_area);

    let outer_block = Block::default()
        .title(" Help — Key Bindings ")
        .borders(Borders::ALL)
        .border_type(BorderType::Double)
        .style(Style::default().bg(Color::Black).fg(Color::White));

    let inner = outer_block.inner(modal_area);
    f.render_widget(outer_block, modal_area);

    let rows: Vec<Line> = bindings
        .iter()
        .map(|(key, desc)| {
            Line::from(vec![
                Span::styled(
                    format!("{:<20}", key),
                    Style::default().fg(Color::Cyan).add_modifier(Modifier::BOLD),
                ),
                Span::raw(*desc),
            ])
        })
        .collect();

    let help_text = Paragraph::new(rows)
        .wrap(ratatui::widgets::Wrap { trim: false });
    f.render_widget(help_text, inner);
}
```

## Task 5: Hamlib Antenna Control via rigctld

**File:** `pancetta-hamlib/src/advanced.rs`, `pancetta-hamlib/src/rigctld.rs`

**Problem:** `get_antenna()`, `set_antenna()`, and `list_antennas()` in `AdvancedRig` (lines 896-912 of advanced.rs) are hardcoded stubs. The rigctld protocol supports `\get_ant` and `\set_ant <ant>` commands, but `AdvancedRig` holds an `Arc<Rig>` (which uses C FFI), not a `RigctldClient` (which has `send_command`).

**Approach:** Since `AdvancedRig` wraps `Arc<Rig>` and `Rig` uses the C Hamlib library (not rigctld), the clean approach is:
1. Add a `pub async fn send_raw_command(&self, cmd: &str) -> Result<String>` to `RigctldClient`
2. Make `AdvancedRig` optionally hold a `RigctldClient` reference for antenna commands
3. For the `Rig` (C FFI) path, keep the stubs — real antenna control via C FFI would require hamlib-sys bindings for `rig_set_ant()` which we don't have yet

**Simpler alternative (chosen):** Since `RigctldClient` already implements `RigControl`, create a parallel `AdvancedRigctld` struct that wraps `RigctldClient` and implements `AdvancedRigControl` with real antenna commands. The existing `AdvancedRig` stays as-is for the C FFI path.

**Even simpler (chosen):** Expose `send_raw_command` on `RigctldClient` (it's just a public wrapper around the private `send_command`), then add a factory method on `AdvancedRig` that optionally takes a `RigctldClient` for antenna operations.

```rust
async fn list_antennas(&self) -> Result<Vec<u32>> {
    if let Some(ref rigctld) = self.rigctld {
        // Try antennas 1-4 and see which ones the rig accepts
        let mut available = Vec::new();
        for ant in 1..=4 {
            match rigctld
                .send_raw_command(&format!("\\set_ant {}", ant))
                .await
            {
                Ok(_) => available.push(ant),
                Err(_) => {} // antenna position not supported
            }
        }
        // Restore original antenna
        if let Some(&first) = available.first() {
            let _ = rigctld
                .send_raw_command(&format!("\\set_ant {}", first))
                .await;
        }
        if available.is_empty() {
            available.push(1); // fallback
        }
        Ok(available)
    } else {
        Ok(vec![1, 2])
    }
}
```
