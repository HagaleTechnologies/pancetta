# FT4 Mode — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use `- [ ]`.
> Each implementer: run `cargo fmt --all` + `cargo check --workspace --all-targets --features transmit` before committing. Never push (controller pushes).

**Goal:** Station-wide selectable FT4 mode (7.5s grid, 4-GFSK) — operator picks FT8 or FT4 at startup; the whole station runs it. **FT8 mode is byte-identical to today** (hard regression invariant). FT4's codec already works (round-trip tests pass); this wires timing + mode-stamping + frequencies.

**Spec:** `docs/superpowers/specs/2026-06-29-ft4-mode-design.md`. **Decisions:** single active mode, startup-selected (runtime toggle = v2); config `[radio] mode`; slot timing parameterized via `_with_period` fns + `active_slot_ns` atomic; mode threaded through `QsoManagerConfig.active_mode`.

**Branch:** `feat/ft4-mode` off main.

## Task 1: Parameterized slot timing in `pancetta-core`
**Files:** `pancetta-core/src/slot.rs`; `pancetta-ft8/src/protocol.rs` (period accessor).
- [ ] In `protocol.rs`: add `pub fn slot_ns(&self) -> i64` to `Protocol` (and/or `ProtocolParams`): `(cycle_duration * 1e9) as i64`. FT8 → 15e9, FT4 → 7.5e9, FT2 → its cycle. Unit-test the three values.
- [ ] In `slot.rs`: for EACH public fn (`current_slot_start`, `SlotParity::of`, `next_slot_start`, `next_slot_with_parity`, `next_audio_start`, `next_phase`) add a `_with_period(…, slot_ns: i64)` variant that takes the period as a param (replace every `SLOT_NS` use inside with `slot_ns`). `next_phase_with_period` asserts `(0..slot_ns).contains(&offset_ns)`. Then **rewrite the existing fns as one-line wrappers** calling the `_with_period` variant with `SLOT_NS`. `SlotParity::of_with_period(t, slot_ns)`.
- [ ] **Regression:** the existing 24 tests in `slot.rs` MUST pass unchanged (do not edit them). Add new tests for `slot_ns = 7_500_000_000`: boundaries at 0/7.5/15s; `SlotParity::of_with_period` alternates each 7.5s (`:00`→Even, `:07.5`→Odd, `:15`→Even); `next_phase_with_period(now, 6.5s, 7.5e9)` within slot; `next_phase_with_period` panics at offset = 7.5s. Keep `PRE_ROLL_NS` as-is.
- [ ] `cargo test -p pancetta-core -p pancetta-ft8 protocol slot`. Commit `feat(core): period-parameterized slot timing (_with_period fns; FT8 fns are SLOT_NS wrappers)`.
