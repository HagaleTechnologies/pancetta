# Neural OSD — Design Spec

## Goal

Replace brute-force OSD trial ordering with a neural network that predicts which LDPC info bits are most likely wrong after BP failure. Reduces OSD trials from ~125K to ~200 while maintaining near-ML decoding performance, enabling a lower parity error gate and recovering more weak signals.

## Current State

- OSD-3 brute-force: tries all `C(91,3) = 125,580` triple-bit-flip combinations
- Gated on ≤3 parity errors remaining after BP (≤2 was too tight)
- Bits ordered by `|LLR|` magnitude (least reliable first) for Gaussian elimination
- ~1.3ms per candidate when OSD triggers
- Message validation (`is_plausible()` + ITU prefix) catches most CRC false positives

## Approach: DIA Model (Decoding Information Aggregation)

Based on "Boosting Ordered Statistics Decoding of Short LDPC Codes with Simple Neural Network Models" (Li et al., IEEE Communications Letters, 2024, arxiv 2404.14165).

Instead of ordering OSD trials by LLR magnitude, train a 4-layer CNN to predict which info bits are actually wrong, using the full trajectory of LLR values across all BP iterations as input. The neural ordering finds the correct bit-flip pattern in ~200 trials instead of ~125K.

### Data Generation

1. Generate random valid FT8 messages using pancetta-ft8's encoder (call via subprocess or reimplement the packing in Python)
2. LDPC-encode each message to 174-bit codewords using the FT8 parity check matrix (hardcoded from `pancetta-ft8/src/ldpc.rs`)
3. BPSK-modulate (simple: bit → ±1.0) and add AWGN noise at SNRs uniformly sampled from -18 to -28 dB
4. Run sum-product BP decoding for 25 iterations, recording the 174 output LLRs after each iteration
5. Check if BP converged (syndrome = 0). Keep only the FAILURES — these are the cases where OSD is needed
6. For each failure, record:
   - Input: LLR trajectory matrix [25 × 174] (f32)
   - Target: error pattern [91] (binary — which info bits are wrong in BP's hard decision vs the known correct codeword)

**Dataset size:** 100K training samples, 10K validation. At -18 to -28 dB, BP fails on roughly 30-70% of trials, so generating ~250K total trials should yield enough failures.

**Note:** The training operates at the LDPC level (174-bit codewords with AWGN), not at the full FT8 audio level. This is correct — the LDPC decoder sees soft LLRs regardless of how they were computed (spectrogram, DFT, AP injection). The neural model learns the code structure, not the channel.

### Model Architecture

Adapted from the paper's DIA model for LDPC(174,91):

```
Input:  [batch, 25, 174]  — LLR trajectory (25 BP iterations × 174 codeword bits)
Conv1D: [batch, 32, 174]  — kernel=3, padding=1, ReLU
Conv2D: [batch, 16, 174]  — kernel=3, padding=1, ReLU
Conv3D: [batch, 1,  174]  — kernel=1, squeeze → [batch, 174]
Linear: [batch, 91]       — sigmoid
Output: [batch, 91]       — probability each info bit is wrong
```

Total parameters:
- Conv1: 25×32×3 + 32 = 2,432
- Conv2: 32×16×3 + 16 = 1,552
- Conv3: 16×1×1 + 1 = 17
- Linear: 174×91 + 91 = 15,925
- **Total: ~19,926 parameters = ~80KB of f32 weights**

### Training

- Loss: Binary cross-entropy (per-bit classification)
- Optimizer: Adam, lr=1e-3, weight decay=1e-4
- Batch size: 256
- Epochs: 50 (early stopping on validation loss)
- Hardware: CPU is sufficient (model is tiny, dataset is small)

### Forward Pass

Pure Rust, no dependencies. The model is 4 layers:

```rust
/// Predict which info bits are most likely wrong after BP failure.
/// Input: LLR values at each of 25 BP iterations for all 174 codeword bits.
/// Output: 91 probabilities (higher = more likely wrong).
pub fn predict_error_bits(trajectory: &[[f32; 174]; 25]) -> [f32; 91] {
    // Layer 1: Conv1D(25→32, kernel=3, pad=1) + ReLU
    let mut h1 = [[0.0f32; 174]; 32];
    for out_ch in 0..32 {
        for pos in 0..174 {
            let mut sum = CONV1_BIAS[out_ch];
            for in_ch in 0..25 {
                for k in 0..3 {
                    let p = pos as isize + k as isize - 1;
                    if p >= 0 && p < 174 {
                        sum += trajectory[in_ch][p as usize]
                            * CONV1_WEIGHT[out_ch * 25 * 3 + in_ch * 3 + k];
                    }
                }
            }
            h1[out_ch][pos] = sum.max(0.0); // ReLU
        }
    }

    // Layer 2: Conv1D(32→16, kernel=3, pad=1) + ReLU
    let mut h2 = [[0.0f32; 174]; 16];
    for out_ch in 0..16 {
        for pos in 0..174 {
            let mut sum = CONV2_BIAS[out_ch];
            for in_ch in 0..32 {
                for k in 0..3 {
                    let p = pos as isize + k as isize - 1;
                    if p >= 0 && p < 174 {
                        sum += h1[in_ch][p as usize]
                            * CONV2_WEIGHT[out_ch * 32 * 3 + in_ch * 3 + k];
                    }
                }
            }
            h2[out_ch][pos] = sum.max(0.0); // ReLU
        }
    }

    // Layer 3: Conv1D(16→1, kernel=1) → squeeze to [174]
    let mut h3 = [0.0f32; 174];
    for pos in 0..174 {
        let mut sum = CONV3_BIAS[0];
        for in_ch in 0..16 {
            sum += h2[in_ch][pos] * CONV3_WEIGHT[in_ch];
        }
        h3[pos] = sum;
    }

    // Layer 4: Linear(174→91) + sigmoid
    let mut output = [0.0f32; 91];
    for i in 0..91 {
        let mut sum = LINEAR_BIAS[i];
        for j in 0..174 {
            sum += h3[j] * LINEAR_WEIGHT[i * 174 + j];
        }
        output[i] = 1.0 / (1.0 + (-sum).exp()); // sigmoid
    }

    output
}
```

Inference cost: ~168K multiply-adds = ~0.01ms on modern hardware.

### OSD Integration

Modify `OsdDecoder::decode()` in `osd.rs` to accept an optional external bit ordering:

```rust
pub fn decode(
    &self,
    llrs: &[f32; 174],
    neural_ordering: Option<&[f32; 91]>,  // neural-predicted error probabilities
) -> Option<BitVec> {
    // If neural_ordering is Some, sort info bits by predicted error probability
    // (descending) instead of by |LLR| magnitude (ascending).
    // The Gaussian elimination and trial loop are unchanged.
}
```

The neural ordering replaces step 2 of OSD (reliability sorting) but steps 3-7 (Gaussian elimination, hard decision, perturbation trials, CRC check) remain identical.

### Decode Flow with Neural OSD

```
1. Extract symbols, compute LLRs
2. Run BP with trajectory collection
3. If BP converges → done
4. If BP fails and parity_errors ≤ threshold:
   a. Run predict_error_bits(trajectory) → 91 probabilities
   b. Call osd.decode(llrs, Some(&probabilities))
   c. OSD uses neural ordering — finds fix in ~200 trials
5. If CRC passes and message is plausible → accept
```

### Parity Error Gate

With neural-guided OSD, we can safely lower the gate from ≤3 to ≤5 parity errors. The neural ordering ensures OSD tries the right bits first, so even with more candidates entering OSD, the trial count stays low (~200) and false positive rate stays controlled. This recovers signals that currently fall through because BP left 4-5 parity errors.

## Future Work: SWA (Sliding Window Assistant)

Deferred from this implementation. The SWA model from the arxiv paper adds early termination prediction during OSD execution:

- **Architecture:** 2-layer dense network. Input = running score statistics within a sliding window of OSD trials. Output = continue/stop binary decision.
- **Benefit:** Reduces average trials from ~200 to ~175 (~12% reduction).
- **Why deferred:** DIA alone captures the primary benefit (smart ordering). SWA requires a second training phase on OSD execution traces. Marginal gain doesn't justify the complexity now.
- **References:**
  - [Boosting OSD of Short LDPC Codes with Neural Networks](https://arxiv.org/abs/2404.14165) — Section IV-B
  - [Boosted Neural Decoders for 6G LDPC](https://arxiv.org/abs/2405.13413) — related neural LDPC work
  - DIA model computational cost: 167.7K FLOPs; SWA: 0.06K FLOPs per frame

## Non-Goals

- Real-time model retraining (the model is trained once offline)
- GPU inference (model is too small to benefit)
- Supporting arbitrary LDPC codes (model is specific to FT8's (174,91) code)
- ONNX or other inference frameworks (pure Rust forward pass is simpler and faster)
