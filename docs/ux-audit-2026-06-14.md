# Pancetta TUI/UX Audit — 2026-06-14

Deep "walk the whole app as a user" audit across 6 flow lanes (input, startup/config/audio,
receive/monitor, DX-hunt/Callers, QSO lifecycle, TX control/freq). Findings below, ranked,
grouped into fix batches. Status: [ ] todo, [x] done.

## Batch 1 — Safety + input correctness (highest operator impact)
- [ ] **TX-policy SAFETY LEAKS**: `p` (TogglePtt) keys the rig with NO policy check — keys carrier
  while DISABLED/after Shift+Q (tui_relay.rs TogglePtt ~921). Shift+T tune also bypasses policy
  (ToggleTune ~836; TuneRequest worker arm tx.rs ~979 has no Disabled hard-mute). FIX: gate PTT
  key-UP and tune on `TxPolicy::allows_any_tx()`; add Disabled hard-mute to the TuneRequest arm.
- [ ] **g→DISABLED doesn't abort in-flight TX** (worker only checks policy at dequeue, tx.rs ~334).
  FIX: cycling to Disabled sets the abort_current_tx flag (like Shift+Q).
- [ ] **Two divergent key handlers**: standalone `pancetta-tui` bin uses stale `App::handle_key_event`
  (app.rs ~659: Esc=quit-no-confirm, j/k=scroll, ONLY place 1-5 bound). Production `TuiRunner` is
  authoritative but binds NO digit keys. FIX: make the bin route through TuiRunner (or delete its
  divergent handler), and add 1-5 panel-jump to TuiRunner::handle_key_event.
- [ ] **No command vs text-input mode**: every letter is both a command and a char; command wins;
  free-text TX unusable; `k`/`s`/`c` fire while "typing" (tui_runner.rs flat match ~578). FIX: explicit
  compose mode (enter via a key, Esc exits; chars→buffer only in compose) so command keys are safe.
- [ ] **`r`/`k` fire globally, not gated to QsoStatus focus** (tui_runner.rs ~758) — `k` aborts the
  selected QSO while scrolling Band Activity. FIX: gate r/k to `ActivePanel::QsoStatus`; echo target
  callsign in the abort feedback.
- [ ] **Cursors not callsign-pinned** → list re-sorts/grows under cursor → Space/Enter/k retarget
  silently. DX Hunter has NO clamp (app.rs ~1418, dx_hunter_scroll positional). Callers clamps the
  override but not the target, and only on Up/Down (app.rs clamp_callers_selection ~1489; not called
  on Home/End/PgUp/PgDn or on decode arrival). FIX: pin both selections to a callsign; re-derive
  index after every list mutation + on decode push.

## Batch 2 — QSO lifecycle visibility + correctness
- [ ] **Watchdog invisible** (the "infinite loop" complaint): call_count/max/deadline never reach the
  TUI; panel shows only `Exchanges: N` (raw msg-log len, frozen during keep-call since rearm doesn't
  append a Sent msg). FIX: add call_count/manual_call_max/watchdog-deadline to ActiveQsoSnapshotItem;
  render `Call 4/10 · stops 3:12` for manual RespondingToCq/SendingReport.
- [ ] **last_tx_text empty for engine TX** (raw_text=String::new() at respond_to_cq_with ~554,
  respond_to_caller ~757, auto-reply ~1083; rearm appends nothing). Panel "TX:" line blank. FIX:
  populate raw_text via MessageExchange::generate_message at creation; append on rearm.
- [ ] **Failure reason never surfaced** (Superseded/Timeout/sender-mismatch all just vanish from the
  snapshot; security-mismatch is log-only). FIX: emit a StatusUpdate carrying QsoFailureReason on
  terminal transitions; brief toast.
- [ ] **Completed never shown** (terminal QSOs filtered from snapshot; ladder_view None). FIX: emit a
  "QSO with X logged" status on QsoCompleted (and/or keep terminal in snapshot ~1 slot with a badge).
- [ ] **Multi-QSO snapshot order non-deterministic** (HashMap.values() unsorted, build_active_qso_snapshot
  ~795) → cursor jumps between QSOs → `k` aborts wrong one. FIX: sort by (start_time, callsign); key
  the TUI qso_cursor on qso_id not position.
- [ ] **CQer completion path unwired**: CallingCq→WaitingForReport, then NO arm out of WaitingForReport
  (determine_state_transition ~1141) → we-CQed QSOs never complete/log, time out. FIX: add CQer forward
  arms; make ladder_view role-aware (CQer WaitingForReport currently blanks).
- [ ] **supersede key case-sensitive** (qsos_by_callsign verbatim) — case mismatch could re-spawn dupes.
  FIX: normalize uppercase in add_callsign_mapping/supersede.
- [ ] **ADIF grid dropped on common close** (SendingReport-close hard-codes grid_square:None ~1230) —
  decoded DX grid never logged. FIX: latch their grid from opening CqResponse → Completed metadata.

## Batch 3 — Config + display polish
- [ ] **Partial pancetta.toml silently reverts ALL config to defaults** (only autonomous/metadata have
  serde(default); skipped-source logged at debug only) → comes up N0CALL/default audio; also breaks the
  device-picker persistence (writes [audio]-only file that then fails to parse). FIX: `#[serde(default)]`
  on the 5 top-level Config sections; warn + TUI-surface on parse-skip; device-picker writes full config.
- [ ] **Speakers-misconfig warning only in log, not TUI** (stream.rs ~483) — the exact bug the operator
  hit. FIX: route to TUI via MessageType::Error + a persistent "TX→system default" badge.
- [ ] **`--list-audio` referenced in messages but doesn't exist**; `test-audio --list` is a stub. FIX:
  implement device enumeration; correct the message text.
- [ ] **No rig-connected indicator** (connect failure logs error! + continues; TUI shows "---" S-meter).
  FIX: rig status badge (Connected/No rig/Polling-failed) from the hamlib poll loop.
- [ ] **Waterfall TX cursor shows manual offset, not live TX freq** (ui/mod.rs ~447 reads
  tx_frequency_offset; should use tx_now_sending.freq_hz when sending) — the 1350→2300 visual bug.
- [ ] **DF/DT columns clipped** (band_activity.rs DF Length(4) for up to "+2800"; DT Length(4) for
  "-15.2"). FIX: widen DF≥6, DT≥5.
- [ ] **TX offset clamp 100–3000 wider than modulator 200–2500** (tui_runner.rs ~647); multi-TX goes
  negative below 200. FIX: clamp to 200–2500.
- [ ] **Late-TX 30s defer invisible** (looks dead); pre-PTT F8 abort leaves QUEUED strip stale; multi-TX
  mislabeled "1 NOW + N-1 QUEUED" though concurrent. FIX: surface deferred/eta in TxItem; clear strip on
  pre-PTT abort; label multi-TX as concurrent.
- [ ] **TX-policy banner never seeded** (TxPolicyStatus defined+consumed but never produced). FIX: emit
  one authoritative TxPolicyStatus at TUI start (or remove the dead variant).
